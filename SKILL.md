# 抑郁自助 · 反刍思维干预

循证自助工具包。核心思路：**情绪来自认知，改变认知就能改变情绪**。

## 跨 session 衔接

本 skill 目录下有 `state.md` 文件，记录了用户的关键状态、已尝试的技术、当前阶段。

- **对话中实时更新**：每次有重要进展（尝试了新方法、状态有变化、有了新发现），立即写入 `state.md`
- **重新加载时**：先读 `state.md`，了解历史状态

**如果你或对方有自杀念头、自伤计划或行为，先做危机干预（见 `references/crisis-resources.md`），再回来做上面的练习。**

## 四个分支

根据用户当下的状态走不同路径。判断入口在下面。

### 分支 A：即时干预（我现在很难受）

用户说"现在很难受/刚崩溃/突然很down/撑不住了"。

步骤：
1. **落地点**：让用户做 3 个 grounding 动作——看 3 样东西、摸 2 样东西、听 1 种声音。完成后再继续。
2. **快速定位**：用户此刻在想什么？帮用户写出脑中自动冒出来的那几句话（自动思维）。
3. **认知扭曲匹配**：从 `references/books.md` 的「10 种认知扭曲」清单中匹配当前自动思维属于哪几种。
4. **思维记录表**：引导用户填一份简易思维记录表（见 `references/daily-exercises.md` 的「思维记录表 mini 版」）。
5. **完成标准**：用户情绪强度自评下降 ≥ 2 分（0-10 分制），或用户说"好一些了"。

### 分支 B：反刍打断（我想太多停不下来）

用户说"反复想同一件事/控制不住/脑子停不下来/rumination"。

步骤：
1. **先区分**：反刍（被动重复想过去）≠ 反思（主动解决问题）。如果是反思，不用打断。
2. **打断技术**：从 `references/rumination.md` 选一个适合当下场景的技术：
   - 正念呼吸（情绪中）
   - 定时担忧法（固定时间专门想，其他时间不想）
   - 视觉打断（情景替换）
   - 行为激活（立刻做一件小事）
3. **重构**：帮用户把反刍内容写成"自动思维 → 认知扭曲 → 理性回应"的格式（调 `references/books.md` 的思维记录表）。
4. **完成标准**：用户能说出一个反刍念头对应的认知扭曲，或者情绪下降 ≥ 1 分。

### 分支 C：每日练习（我想系统改善）

用户说"想系统改善/每天做点什么/长期调理"。

步骤：
1. **评估起点**：用户填 Burns 抑郁检查清单（`books/feeling-good.md` 第 2 章有完整量表），确定严重程度。
2. **选一个练习**：从 `references/daily-exercises.md` 选一个符合当下状态的练习：
   - 状态很差 → 行为激活（每天做一件小事）
   - 自我批评严重 → 三栏技巧 / 语言柔道
   - 完美主义 → 反完美主义表
   - 焦虑/担忧 → 定时担忧法 + 正念呼吸
3. **定最小行动**：用户承诺每天做 X 分钟/做 X 件事。不超过 15 分钟。
4. **完成标准**：用户定好一个可执行的每日练习。

### 分支 D：书籍导航（我想看书/了解理论）

用户问书/理论/某本书的内容/某个方法。

步骤：
1. **定位场景**：用户当前状态 + 想解决的问题（反刍/自卑/拖延/焦虑/完美主义/关系问题）。
2. **匹配书籍**：从 `references/books.md` 找到对应的书和章节。
3. **提取关键方法**：给出 1-2 个可以直接用的技巧（不是理论，是可操作步骤）。
4. **引用原文**：如果方法来自某本书的全文，直接从 `books/` 下对应的文件摘录原文。
5. **完成标准**：用户拿到一个可操作的技巧。

## 核心工具速查

最常用的 3 个工具，记在脑子里：

**1. 三栏技巧（Three-Column Technique）**
```
自动思维（原话） | 认知扭曲（哪种） | 理性回应（更客观的话）
```
用户写出第 1 栏 → 你帮标第 2 栏 → 用户自己想第 3 栏。

**2. 10 种认知扭曲**（完整定义见 `references/books.md`）
1. 非黑即白 / 全有全无
2. 过度概括（一件事代表全部）
3. 心理过滤（只看见负面）
4. 否定正面（正面不算数）
5. 读心术（以为别人怎么想你）
6. 预言家（一定很糟）
7. 灾难化（往最坏处想）
8. 情绪推理（感觉糟所以就是糟）
9. 应该陈述（我应该/我必须）
10. 贴标签（我是废物/他是坏人）

**3. 反刍打断三问**
- "这件事我还能改变什么？" → 能 → 列行动计划；不能 → 下一步
- "如果朋友遇到同样的事，我会怎么跟他说？"
- "一周后/一个月后，这件事还重要吗？"

## 加新书流程

新增一本书到 skill 时，按以下步骤更新：

1. **全文**：epub → markdown 放到 `books/<英文书名>.md`（用 pandoc 转）
2. **方法总结**：在 `references/books.md` 末尾新增一条，格式参照已有的 7 本书——写作者、核心方法、适用场景、关键技巧
3. **练习关联**：如果新书有可操作的每日练习，加到 `references/daily-exercises.md`
4. **反刍/危机**：如果新书有反刍或危机干预相关内容，补充到对应 reference 文件
5. **路径检查**：确保 SKILL.md 中引用新书的地方路径是 `books/<文件名>` 而不是旧路径

### 示例（加第 3 本书）
```
books/蛤蟆先生去看心理医生.md
references/books.md  → 新增一条「#. 《蛤蟆先生去看心理医生》」
state.md → 如果用户读了，写下当前在读的书
```

## 与其它技能的搭配

- `warmth` → 如果用户是写温暖向故事但不是抑郁自助
- `emotion` → 如果用户是写作相关而不是心理自助
- `human` → 如果用户想让文字更像人而不是做心理调节
